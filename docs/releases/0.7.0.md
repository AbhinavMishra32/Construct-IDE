# Construct 0.7.0

## Highlights

- The desktop shell now has a more focused workspace experience: a compact circular Home control, responsive inspector motion, and a settings layout that does not compete with project navigation.
- Flow chat can move between panel, expanded, and hidden states directly from the embedded agent surface while preserving the live session and workspace context.
- The Studio sidebar supports project creation, sorting, pinning, archiving, restored project visibility, and durable sidebar preferences.
- Profile settings are now backed by the Rust-owned application store. They include a persistent identity, activity summary, project mix, and a concise current AI setup view.
- Native command failures are normalized at the Tauri bridge, so renderer failures surface a useful error message instead of serialized object output.
- Project summaries now carry creation time consistently across Rust and compatibility storage paths, enabling stable project ordering.

## Release integrity

- All application version sources are aligned at `0.7.0`: root package metadata, app package metadata, Cargo, and Tauri bundle configuration.
- Release preflight now asserts Cargo/package version alignment.
- Canary workflow versioning updates Cargo alongside the JavaScript and Tauri manifests, preventing a mixed-version packaged binary.
- The release workflow retains targeted Opaline checkout, no-build preflight before the packaging matrix, and idempotent GitHub asset upload behavior.

## Verification

- `pnpm verify:no-build`
- `cargo check --manifest-path app/src-tauri/Cargo.toml`
- `cargo test --manifest-path app/src-tauri/Cargo.toml --lib`
- `git diff --check`
