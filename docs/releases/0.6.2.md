# 0.6.2

This release tracks the current `main` branch and folds in the latest shell, graph, auth, styling, and release-pipeline work.

## What's Changed

- Refreshed the bundled Opaline submodule pointer so the desktop app carries the latest shared shell and graph chrome
- Regenerated the release app icons and kept the platform packaging artwork in sync with the current brand surface
- Added reusable auth alert and logo chrome so sign-in and recovery screens share the same visual language
- Tightened the sign-in and recovery flows so auth states and error handling stay clearer under the new chrome
- Refined the window manager controls for the shell header
- Reorganized the application shell so the primary desktop surfaces read more cleanly
- Rebalanced dashboard navigation chrome to better match the rest of the app
- Reworked file-tree row sizing to improve density and scanability
- Deepened Flow workspace and knowledge-state handling so concept and session surfaces stay more coherent
- Updated the global knowledge graph surface to use the shared 3D controller and a cleaner backdrop
- Refined the concept card graph entry so it now opens the full global concept graph, highlights the current concept, and reuses the same selection and camera behavior as the main concepts page
- Tuned the renderer surface palette to better fit the dark product UI
- Added test coverage for the settings and interaction surfaces touched by the shell and auth changes
- Locked knowledge graph and concept-card behavior with tests so the shared graph controller stays stable

## Highlights

- The concept card graph is now the global concept graph, not a project-only subset
- The current concept is highlighted immediately when the graph opens
- The card now reuses the same controller as the main concepts surface, so selection and camera behavior stay consistent
- The release metadata is version-aligned across the root package and desktop app package at `0.6.2`
