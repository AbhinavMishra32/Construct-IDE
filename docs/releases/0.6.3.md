# 0.6.3

This release fixes Windows desktop windowing so Construct behaves like a real draggable native app window while keeping the macOS window configuration unchanged.

## What's Changed

- Added a Windows-only Electron titlebar overlay so native minimize, maximize, and close controls remain available with custom chrome
- Switched the Windows window surface away from unsupported transparent-window behavior and onto the Windows Mica background material
- Added Windows-scoped drag and no-drag regions for the shell header so the window can be moved without breaking normal controls
- Kept the existing macOS `hiddenInset` titlebar, traffic-light placement, vibrancy, transparency, and visual effect settings untouched
- Bumped the desktop release metadata to `0.6.3`

## Highlights

- Windows users can drag the app from the shell header again
- Windows now gets native controls with modern transparent-style system material
- macOS users should see no behavioral or visual change from `0.6.2`
