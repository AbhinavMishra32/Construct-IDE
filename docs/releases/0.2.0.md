# Construct 0.2.0

Construct 0.2.0 brings the learning conversation into the project itself. Construct Interact can now investigate the authored tape, learner history, project context, and terminal evidence before responding, while showing the activity that informed each answer.

## Highlights

- Full multi-step Construct Interact agent runs powered by Mastra
- Agent-chosen tools for authored steps, blocks, concepts, references, learner history, project files, and terminal output
- Expandable per-turn activity traces that remain available after reload
- Source-aware answers that distinguish lesson text from concept-card and reference-card text
- Independent introduced, opened, and saved concept state, including unsaved concept engagement
- Contextual actions for opening concepts, files, authored steps, and validated live learning steps
- Consistent OpenRouter and OpenAI-compatible provider/model selection across Construct agents
- Codex-style conversation geometry with clear user messages, unboxed assistant replies, compact metadata, and a sticky composer
- `tape-0.4.2` support while retaining compatibility with earlier Construct tapes

## Tape 0.4.2

In `tape-0.4.2`, the `steps`, `concepts`, `references`, and `files` values in `::resources` are discovery hints. Construct gives the agent the project id, block id, learner message, and scoped tools. The agent decides what to inspect and in what order, then returns a typed learning result with optional actions and state updates.

The app records the tools and bounded results used by each turn. This gives learners an inspectable activity trail without exposing private model chain-of-thought.
