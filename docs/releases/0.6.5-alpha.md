# 0.6.5-alpha

Construct 0.6.5-alpha tightens the Construct Cloud model path, simplifies agent model selection, fixes Knowledge Web loading, and refreshes the release handoff so GitHub can package and publish the prerelease from the pushed tag.

## What's Changed

- Fixed Knowledge Web startup so saved concepts render into the graph even when the pane has not reported stable layout dimensions yet
- Replaced the Knowledge Web text loader with the shared Construct silhouette shimmer and removed the dark-mode glow layer from the graph background
- Extended the darker Construct silhouette shimmer to app startup and account/user loading states
- Added Construct Cloud model preflight before model calls, with explicit catalog lookup errors when the chosen model is not available
- Renamed hosted-compute UI copy to Construct Cloud across settings, model picker, status, and account surfaces
- Simplified AI settings around one Construct agent model instead of per-feature overrides
- Made Flow composer model selection read-only from the composer, with model changes routed through AI settings
- Updated status and composer model labels so Construct Cloud models are represented as first-class provider choices
- Refreshed dependency lock state for the desktop workspace
- Updated GitHub release publishing so alpha prereleases use the checked-in changelog when the tag is pushed
- Bumped the desktop release metadata to `0.6.5-alpha`

## Highlights

- Knowledge Web should no longer get stuck on `Loading knowledge web...`
- Construct Cloud model failures now explain the actual unavailable-model boundary before an agent run starts
- The release tag can be pushed and GitHub Actions will handle packaging and publishing with these notes
