# Construct 0.3.0 — Flow Projects

Construct 0.3.0 introduces **Flow Projects**, a new project type for open-ended, agent-driven coding mentorship. Flow replaces rigid lesson structures with a natural tool-calling coding mentor that inspects files, searches code, creates practice tasks, runs terminals, and adapts to the learner — all in a real workspace.

This release also overhauls the AI provider system with a managed LiteLLM proxy, OpenCode Zen support, provider-level logging, and a redesigned AI settings panel.

## Highlights

### Construct Flow Projects

- New **Flow** project type — loose, natural, tool-call-native coding mentor workspace (no tape steps or rigid lesson graphs)
- **Flow Memory** — durable project memory stored as markdown files (`research.md`, `project.md`, `path.md`, `learner.md`) in `.construct/`
- **Flow Research Agent** — runs at project creation to research the domain/technology and create `research.md` using internet search
- **Main Flow Agent** — single coding mentor agent with real workspace powers: file inspection, code search, LSP analysis, practice tasks, terminal commands, concept management, and patch proposals
- **Practice Task tool** — creates real coding tasks with baseline capture, learner submission, and compact diff return
- **Flow Workspace** — dedicated UI with left file tree, center editor/terminal, right agent conversation with streaming tool activity
- **Flow Memory viewer** — inspect/read flow memory files in project settings
- **Streaming ordered agent reasoning** — tool calls, results, and reasoning render in real-time inside agent traces
- **Shared Construct Protocol tools** — `flow-memory-read`, `flow-memory-update`, `flow-memory-ensure`, `internet-search`, `ask-user`, `practice-task`, `find-files`, `search-content`, `view`, `lsp-inspect`, `focus-code`, `edit-propose-patch`, `edit-apply-patch`, `edit-write-file`, `edit-replace`, `workspace-diff`, `run-terminal-command`, `terminal-latest`, `concept-create`, `concept-update`, `concept-find`, `learning-record`, `checkpoint-create`

### AI Provider Overhaul

- **OpenCode Zen** replaces the old local `opencode` provider — new remote AI provider via OpenCode Zen API gateway
- **Managed LiteLLM proxy server** — Construct can now spawn and manage its own LiteLLM child process with Start/Stop in settings, auto-install via pip3, and graceful shutdown
- **Redesigned AI settings panel** — grouped layout: Provider → Credentials → Model → Per-feature overrides → Managed server with new `ProviderModelPicker` component
- **Provider-level logging** — new `ProviderLogService` emits structured logs per provider (OpenAI, OpenRouter, OpenCode Zen, GitHub Copilot) to the output panel
- **Restructured output panel** — main dropdown groups by AI/Agents/LSP/System with sub-select for individual channels

### UI & Shell

- **Compact VSCode-style status bar** at the bottom edge showing provider status, LiteLLM server state, and quick actions
- **Polished shell interactions** — refined terminal panel, sidecar container scroll bounds, and consistent UI across all panels
- **Flow tabs restored** — tab navigation restored in the workspace with proper shell alignment

### CI & Infrastructure

- **Automated canary pre-releases** — manual workflow dispatch without a tag produces a dated canary build (`0.2.1-canary.YYYYMMDD.sha`)
- **Submodule checkout** for typechecking and builds in CI
- **Cross-platform packaging fixes** — Windows MAX_PATH workaround, Linux deb metadata, recursive asset discovery, cross-platform icon generation
- **TypeScript type unification** — resolved `@types/react` / `@types/react-dom` version mismatch
