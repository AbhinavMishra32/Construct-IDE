# Construct 0.6.7-alpha

## Changes

- Flow now persists assistant reply timeline parts even when a question/tool call is present, preventing prior agent text from disappearing when a tracked question pauses the run.
- Practice-task review is evidence-based instead of diff-only: terminal-created work and command-only subtasks can be reviewed and completed from workspace evidence even when the formal submission has no file diff.
- Task submission UI is cleaner and subtask-scoped, with per-subtask submit controls and no learner-facing diff dump for review requests.
- Ask-question supports optional code-answer defaults, agent-controlled material hiding, and answer skipping only when explicitly allowed.
- Concept material can be hidden during recall checks; open concept cards blur and the Concepts sidebar button is disabled until the question is answered.
- File-tree folder expansion has explicit Radix accordion keyframes again, restoring smooth directory expand/collapse motion.
- Terminal output now batches PTY data before IPC and batches renderer writes before xterm rendering for smoother high-throughput output.
- Language Server support is registry-driven across TypeScript/JavaScript, Python, Rust, Go, Java, C/C++, C#, HTML, CSS, and JSON, with per-language install/start status in Settings.
- Rust language-server installs now route through the selected Rust toolchain or Homebrew fallback instead of collapsing into the generic npm installer, so the install button surfaces a real dependency error instead of hanging on a missing `rustup`.
- GitHub release publishing now clears prerelease state for tag-pushed alpha releases and publishes the release title as the version string, so `v0.6.7-alpha` can ship as the latest official release without the `Construct` prefix.
- Code rendering and settings polish continue the Construct code-theme and editor integration pass from this alpha cycle.
